# Описание проекта
Использован стандрартный шаблон для проекта, который включает в себя:

- react 18
- typescript
- jest
- eslint
- prettier
- husky
- zustand

Учтены все замечания:
- минимум комментариев (самодокументируемый код)
- убраны все лишние настройки из репозитория (Storybook etc)
- большинство компонентов и страниц переделаны на семантическую верстку (для SEO)
- декомпозированы до разумного предела все сущности

Важный аспект по функционалу:

Приложение тестировалось с включенным VPN, т.к. из моей локации АПИ, указанное в ТЗ, недоступно.

Поиск настроен согласно АПИ https://dummyjson.com/docs/products, но есть ньюанс. Я тестировал различные запросы, с 
разными параметрами (например, чтобы select был по title), однако бекенд выдает несколько странные данные. Например,
при поиске pow мы ожидаем списка товаров, в которых в заголовке так или иначе встречается эта строка. К сожалению,
это не совсем так. Поэтому увы, но в таблицу рендерится список продуктов, полученный из АПИ, но чтобы было все корректно, я вижу как вариант такой фикс: получить так или иначе весь список, а затем его корректно фильтровать уже на клиенте. На само ТЗ это никак не влияет, описываю для того, чтобы было ясно - кейс продуман, замечен, и далее тема эскалируется для дальнейшего обсуждения.
